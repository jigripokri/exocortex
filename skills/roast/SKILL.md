# /roast

User wants to deploy a roast in a group chat. You write it. They send it.

## Recipe

1. **Identify the target** (a person, a recent message they sent, a running joke).

2. **Read the last 30–50 messages in the group chat.** Critical step — the roast has to match the group's existing tone and reference shared material.

3. **Identify the roastable seam:**
   - A recent flex that backfired
   - A predictable behavior pattern of the target
   - A callback to an old joke at their expense
   - A new piece of information that contradicts their stated position
   - A comparison that punctures their self-image

4. **Calibrate intensity** to the group:
   - **Light**: gentle ribbing, lots of love underneath
   - **Medium**: pointed, but with a clear out (target can laugh)
   - **Hot**: no quarter, but stays inside group norms

   Default to **Medium** unless the group's recent tone is clearly Hot.

5. **Generate 2–3 options.** Different angles or escalation levels.

6. **Show the options. Wait for "send X."**

7. **On approval, send via WhatsApp/Slack/wherever the group lives.**

## Roast rules (hard limits)

- **No body-shaming.** Ever.
- **No real shots at the target's family** unless they've made a self-deprecating reference first.
- **No racism, sexism, homophobia, transphobia.** Even when group "ironically" jokes that way.
- **No quoting from carry-on.md.** This skill cannot reference any private file.
- **No bringing up the target's actual current pain** (job loss, health, relationship). Roasts are for ambient material, not actual wounds.
- **One landed roast > three forced ones.** If nothing's good enough, say so. "Nothing roastable in this thread right now — wait for them to set something up."

## Voice rules

Mirror the group's vocabulary, slang, and inside jokes. A good roast feels like the group wrote it, not an outsider.

If the user wants the AI's voice to come through, that's a different skill — this one is invisible.
